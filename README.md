# 道藏 · 经部（在线读本）

本站为多部典籍提供在线阅读。每部书有独立目录与章节页：原文可切换**拼音**辅助；有释义的经书在释义旁提供**释义风格**切换（各书条目多少以数据为准）。适合慢慢读、反复读。

## 路由说明

- **`/jing`**：经书总目，择一书进入。
- **`/jing/daodejing`**、**`/jing/yinfujing`**、**`/jing/zhuangzi`**：对应书的阅读器；章节通过查询参数 **`?chapter=<章号>`** 打开某一章（与浏览器前进/后退一致）。
- **`/`**：默认进入 **`/jing`**；若旧链接带 **`/?chapter=…`**，会自动跳到 **`/jing/daodejing?chapter=…`**，以兼容早期仅道德经时的入口。

## 可以怎么用

- **目录**：每部书内总览全部章节，卡片上可看到阅读状态与单章累计时长等。
- **释义**：在展示释义的经书中，可按条目切换不同角度的说明；庄子等书在数据中为简式释义，界面会以实际字段为准。
- **夜览与字号**：阅读器右上角可切换夜览、调整正文字号（偏好记在本地）。
- **离线阅读**：部署环境（例如 GitHub Pages）下会注册 Service Worker，**在已成功打开过站点之后**，常见浏览器可在无网络或网络不稳时继续浏览已缓存的页面与资源。首次访问、大版本更新后仍需要联网加载；若清理了站点数据，需再次在线打开以恢复缓存。
- **进度与时光**：**每一部书单独**记住你读过哪些章、在某一章上大概花了多少时间。这些信息只保存在**你当前使用的浏览器**里，不会上传到服务器；换浏览器或清除站点数据会相当于该书重新开始累计。

## 关于「悟道」与进度

每一章都有一条从心入门到渐深领会的小径，名字从 **叩门、涉溪、观象**，到 **闻道、存思、入微、悟道**。你可以按自己的感受，在阅读时调整当前章节所处的阶段——这没有标准答案，只是帮你标记「我和这一章相处到哪一步了」。

当你把某一章调到 **闻道** 及之后的阶段时，这一章会计入该书「已读」；若改回更早的阶段，则会从已读里撤下。顶上的阅读进度百分比，按 **已读章数 ÷ 该书总章数** 计算。

打开过、但还没调到「闻道」及之后的章节，会视作「在读」。**阅读时长**只在您停留在某一章的正文页、并且浏览器页签处于前台时累计；回到目录、切换到别的标签页或关掉页面，当前这一章的计时会先记下来，下次再进同一章会接着累加。

## 一起把文字订正得更准

经文与释义在流传与整理中难免有讹误或表述可改进之处。若你发现某章原文、某条释义不够准确，欢迎**在线参与订正**：

- 在阅读页释义正文旁，可点击**修改（铅笔）图标**，跳转到本仓库在 GitHub 上对应经书的源文件：**`src/books/<经书 slug>/chapters.json`**（例如 `daodejing`、`yinfujing`、`zhuangzi`）。
- 修改后向本仓库提交 **Pull Request（PR）**；维护者审阅合并后，站点会随部署更新，方便更多读者读到修订后的内容。

本地开发时，若配置了 **`VITE_GITHUB_REPO`** 或在 `package.json` 中填写了 **`repository`**，铅笔链接会更容易解析到正确的仓库地址（与 GitHub Actions 部署时的环境变量一致）。

这既适合改错别字、标点与断句，也适合在充分依据的前提下补充、调整释义表述。提交前请尽量说明出处或理由，便于审阅。

## 本地开发

```bash
npm install
npm run dev
```

```bash
npm run build   # 类型检查 + Vite 生产构建
npm run lint
npm run preview # 预览构建结果
npm run deploy  # 构建并执行 scripts/deploy.sh（若你自用该脚本上传）
```

GitHub Pages 一般由仓库 **Actions** 部署；公开地址为 **`https://<用户名>.github.io/<仓库名>/`**（仓库名为 `daojing` 时即 `/daojing/`）。CI 会按仓库名自动设置 **`VITE_BASE_PATH`**，本地若需模拟子路径可手动指定（见 `.github/workflows`）。
